# 安卓移植计划

## 摘要
基于当前仓库只有 dist 静态产物、且目标是“安卓专用 + 原生体验优先”，本项目不建议继续套 WebView 壳。更合适的路线是：

- 使用 Kotlin + Jetpack Compose 新建原生 Android 项目
- 先把网页里的核心计算逻辑抽成纯 Kotlin 计算层
- 第一版只做“参数输入 + 结果展示”，图表后置
- 日常预览以 Compose Preview + Live Edit + 模拟器/真机 为主
- release 用 Android Studio / Gradle 生成签名 APK，适合自己分发

当前仓库可作为“规则与公式参考”，因为 sourcemap 显示原项目大概率来自 Vue 2 + Element UI + ECharts，且核心公式函数可以从打包产物中恢复。

## 关键实现

### 1. 项目结构与迁移方式
- 新建 Android Studio 工程，技术栈固定为 Kotlin + Jetpack Compose + Material 3
- 分成 3 层：
  - domain：齿比、踏频、功率、坡度等纯计算逻辑
  - ui：Compose 页面、输入组件、结果卡片
  - data：牙盘/飞轮/轮径/姿势等预设常量
- 不直接复用现有网页 UI；只复用业务规则、默认参数、公式、文案
- `info.md` 中的说明文字迁到应用内“说明/关于”页
- 第一版不做复杂图表、不做 WebView、不保留原网页的 ECharts 时间轴/缩放交互

### 2. 先做的页面与交互
- 首页只保留一个主流程：
  - 选择牙盘/飞轮/轮径/体重/踏频/坡度/姿势等参数
  - 点击计算或输入即算
  - 展示结果卡片
- 第一版结果建议固定为：
  - 齿比列表
  - 典型踏频对应车速
  - 指定坡度/姿势下目标车速所需功率
  - 最小齿比爬坡能力
- 双盘/单盘逻辑在输入层直接支持
- 结果页先用列表、表格、卡片表达；图表放到第二阶段
- 若第二阶段补图表，默认选 Compose 友好的图表库；只有在必须高度还原原网页曲线交互时，再评估混合方案

### 3. 方便预览每次修改的工作流
- 日常最快预览：
  - `@Preview` 预览单个组件和整页状态
  - 为每个页面准备 PreviewData，不要在预览里直接依赖 ViewModel 或真实数据源
- 联调预览：
  - Android Studio Live Edit 连模拟器或真机
  - 常驻一个中端机型模拟器，例如 Pixel 7 / Android 14+
- 真机验证：
  - 保留 1 台安卓真机，用无线调试快速安装
- 建议固定三层验证节奏：
  - 改样式/布局：先看 Compose Preview
  - 改页面行为：再看 Live Edit
  - 改计算逻辑/安装包行为：最后跑模拟器或真机完整流程
- 为避免每次都手填参数，首页加入“加载示例配置”按钮，便于回归测试

### 4. release 包构建方式
- 当前目标是自己分发，主产物定为：signed release APK
- 构建链固定：
  - 开发构建：debug APK
  - 对外分发：release APK
- 签名方案：
  - 使用 Android Studio 生成 keystore
  - 签名信息写入本地 `keystore.properties` 或环境变量，不提交仓库
- 命令行构建默认流程：
  - `./gradlew assembleDebug`
  - `./gradlew assembleRelease`
- 发布前最少检查：
  - `minifyEnabled` 是否开启
  - `versionCode` / `versionName` 是否递增
  - 应用图标、包名、签名是否正确
  - 在未安装开发版的干净设备上安装一次 release APK
- 如果后续要上架，再补：
  - `bundleRelease` 产出 AAB
  - 应用商店所需隐私、截图、签名与渠道配置

## 公共接口 / 类型约定
- 输入模型：
  - `BikeConfig`
  - `RiderConfig`
  - `EnvironmentConfig`
  - `CalculationRequest`
- 输出模型：
  - `GearRatioResult`
  - `CadenceSpeedResult`
  - `PowerSpeedResult`
  - `ClimbCapabilityResult`
  - `CalculationReport`
- 计算层要求：
  - 全部纯函数
  - 不依赖 Android API
  - 舍入规则、单位换算、默认参数集中管理
- UI 层只接收已经整理好的 UiState

## 测试计划
- 单元测试：
  - 单盘/双盘齿比计算
  - 踏频与车速换算
  - 功率-车速计算
  - 最小齿比爬坡能力
  - 边界值：最小踏频、最大踏频、负坡度、空数组/非法输入
- 对照测试：
  - 选 5 到 10 组网页已有参数，和当前网页结果做人工对照，误差范围先定到可接受的小数位
- UI 测试：
  - 小屏手机不横向溢出
  - 旋转屏幕状态不丢失
  - 示例配置一键加载正常
- 发布测试：
  - release APK 可安装
  - 冷启动正常
  - 核心计算流程离线可用

## 默认假设
- 当前目录没有可直接维护的 Vue 源码，只有发布产物；迁移时以 sourcemap 和页面行为为参考
- 第一版目标是“可稳定使用的原生安卓计算器”，不是“网页 1:1 复刻”
- 第一版默认离线可用，不依赖后端
- 第一版默认不做复杂图表；如果后面确认要补图表，再开第二阶段
- release 以签名 APK 为主，不优先做商店上架包
